== YOUR ANALYSIS TASK ==
Interpret the fatigue signals above and the raw context to determine if this creative is fatigued, trending toward fatigue, or still fresh.

Key signals to evaluate:
- ctr_slope_7d: slope of CTR over last 7 days. This field is usually a rate, so -0.001 means roughly -0.10 percentage points/day. Do not treat tiny negative values such as -0.0002 as catastrophic by themselves.
- ctr_decay_pct: relative decline from first_7d_ctr to last_7d_ctr. Below -0.5 = severe decay
- creative_status: pre-labelled status (fatigued / stable / top_performer / underperformer)
- fatigue_day: day when fatigue was first detected (if available)
- active_days: total days this creative has been running
- creative_age_days: age of creative in days

Decision rules:
- PAUSE: only if fatigue is paired with clear harm: poor return, weak current performance versus campaign peers, or high spend being wasted. PAUSE means continuing this exact creative is actively harmful.
- PIVOT: use this when the creative has useful historical signal but recent fatigue or audience wear-out suggests the current execution should be refreshed. If the concept worked before but is now tired, prefer PIVOT over PAUSE.
- TEST_NEXT: active_days < 7 (too early to detect fatigue)
- SCALE: no decay signal, creative_status = "top_performer" or "stable" with strong recent performance

Calibration:
- Do NOT vote PAUSE just because ctr_decay_pct is below -0.5. Lifetime decay is common in mobile ads; if return or install volume is still strong, vote PIVOT and explain what should be refreshed.
- If creative_status is "top_performer" and fatigue is the only concern, vote PIVOT or SCALE with monitoring, not PAUSE.
- If the evidence is uncertain, vote TEST_NEXT, not PAUSE.
